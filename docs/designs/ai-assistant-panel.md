## Vision

### 10x Check
The 10x version is a JazeOS that barely needs you to type. Gmail watches your inbox, calendar watches your schedule, bank emails auto-propose entries. The dashboard becomes a confirmation queue. Chat exists as fallback for manual entries and corrections. The app runs itself while you tap "confirm" a few times a day.

## Key Implementation Decisions
- AI Package: Laravel AI SDK (`laravel/ai`, first-party, ships with Laravel 13)
- Chat Endpoint: Separate JSON API at `/api/assistant/*` (not Inertia)
- Branch: separate feature branch (user requested, overrides default main-branch workflow)
- "Create if not found" behavior: LLM creates new record if referenced entity doesn't exist, with user confirmation via ProposedAction card
- Abort fetch on navigate-away: use AbortController in React to cancel in-flight LLM requests when user navigates
- Inertia state sync: after confirming a proposed action, trigger `router.reload()` to refresh current page data (per outside voice)
- Undo mechanism DROPPED: too complex (soft delete retrofit across models). Action history feed shows recent entries without undo.

## Cherry-Pick Specs

### 1. Smart Suggestions (Quick Entry Bar)
- On keydown in quick entry bar, debounce 300ms, then query last 20 assistant-created entries matching current input prefix
- Display top 5 suggestions as dropdown below input (shadcn Popover)
- Suggestions format: "Paid $X at [vendor] ([category])" based on recent expense patterns
- Tab to accept suggestion, fills input, user presses Enter to submit
- Data source: `assistant_messages` table where `proposed_actions` was confirmed, grouped by action type

### 2. Action History Feed
- New dashboard section: "Recently logged via Assistant" showing last 10 confirmed actions
- Each entry shows: icon (module type), summary text, timestamp, link to view/edit record
- No undo mechanism (dropped per outside voice review: soft delete retrofit too complex)
- Users can navigate to the record and delete from the normal UI if needed
- Data source: `assistant_messages` table where `proposed_actions` has `confirmed_at` set

### 3. Context-Aware Page Actions
- Current page detected via Inertia's `usePage().component` (e.g., "Subscriptions/Index")
- Mapping table:

| Page Component | Placeholder Text | Extra Context Loaded |
|---------------|-----------------|---------------------|
| Dashboard | "What happened?" | All modules (default) |
| Subscriptions/* | "Add or manage a subscription..." | All subscriptions with details |
| Expenses/* | "Log an expense..." | Recent expenses, budget categories |
| JobApplications/* | "Log an application or interview..." | Open applications with statuses |
| Investments/* | "Record a transaction..." | Active investments |
| UtilityBills/* | "Log a bill payment..." | Active utility accounts |
| Default | "What happened?" | All modules (default) |

- Context passed to backend via `X-Current-Page` header on fetch calls

### 4. Natural Language Queries
Additional read-only tool definitions:

| Tool | Description | Returns |
|------|------------|---------|
| `queryExpenses(filters)` | Search expenses by date range, category, vendor, amount | List of matching expenses |
| `querySubscriptions(filters)` | Search subscriptions by name, status, cost range | List of matching subscriptions |
| `summarizeSpending(period)` | Aggregate spending by category for a time period | Summary with totals |
| `getUpcoming(days)` | Bills due, interviews scheduled, contracts expiring in N days | Prioritized list |
| `queryJobApplications(filters)` | Search by company, status, date range | List of matching applications |

- Query results returned as structured data, formatted as natural language by the LLM
- Guardrails: all queries scoped to current tenant + user (same as existing controllers)
- Large result sets: limit to 20 results, tell user "showing first 20 of N results"
- Cost: read-only queries use smaller context (no need to load full module context for writes)

### 5. Morning Briefing
- Built-in command: user types "briefing" or "what's happening today?"
- Tool definition: `generateBriefing()` with no parameters
- Composes from existing dashboard data:
  - Bills due this week (amount, count)
  - Subscription renewals in next 7 days
  - Upcoming interviews (next 7 days)
  - Budget utilization percentage
  - Overdue items requiring action
- Response is natural language, not structured cards
- Can be cached for 1 hour (dashboard data doesn't change that fast)
